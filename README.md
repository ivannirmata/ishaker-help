# iShaker Help — draft

Черновик справочных страниц для операторов iShaker (RU + EN). Собран как основа для будущей публикации на публичной вики; **это черновик для внутреннего ревью**, не финальный контент.

## Что внутри

- `index.html` — оглавление: 30 тем по приоритетам, у каждой ссылки RU / EN.
- `ru/`, `en/` — по одной странице на тему на каждом языке (слаг одинаковый в обеих папках).
- `assets/style.css` — оформление.
- `build.py` — генератор сайта: весь контент лежит здесь, `python3 build.py` пересобирает все страницы.

## Статус контента

- Все **30 тем** — черновой полный текст RU + EN.
- Осталось: вычитать, добавить скриншоты с реальных машин, сверить точные названия вкладок (особенно сервисные меню Touch 2 / Shaker S — помечено в тексте).

## Как писать статьи

Гайд для наполнения справочника (где брать материал и изображения, тон, чего не публиковать, технические шаги) — **[AUTHORING.md](AUTHORING.md)**.

## Как продолжить

1. Открыть `index.html` в браузере (или включить GitHub Pages: Settings → Pages → branch `main` → `/ (root)`).
2. Править текст в `build.py` (список `ARTICLES`) и запускать `python3 build.py`, либо редактировать готовый HTML напрямую.
3. Скриншоты снимать на реальной машине соответствующей модели, класть в `assets/` и вставлять в страницы.
4. Правило контента: страница = либо «что оператор делает сам», либо «как/почему это работает». Внутренние процедуры (доступы, командная кухня) на публичные страницы не выносить.

## Предпросмотр

Открыть `index.html` локально, либо через GitHub Pages — `https://ivannirmata.github.io/ishaker-help/`.
